# Design Principles

1. **Log in seconds.** Every screen between the user and a logged meal must justify itself. Scan → confirm is the golden path (≤ 3 taps).
2. **Your macros, your way.** Focus on one macro or see all four; count up or down. Preferences persist, never reset.
3. **Honest about allergens.** Red/green/grey (contains/free/unknown). Grey is a first-class state — we never fake certainty.
4. **Natural portions.** "1 burger (150 g)", never "1 gram". Defaults respect how people actually eat. Fractional counts (½ pack) are allowed.
5. **Meals like real life.** Breakfast → lunch → dinner backbone, snacks wherever they happen.
6. **Community-visible, not community-noisy.** Versions and likes support data quality quietly; no social pressure in the diary.
7. **Bilingual as equals.** Dutch and English are both first-class; no half-translated screens.
8. **Free means free.** No upsell dark patterns, no locked buttons, no ads. Donations live in About, nowhere else.
9. **Calm visual language.** Modern, clean, high-contrast; celebrates consistency (streaks of logged days) rather than shaming gaps.
