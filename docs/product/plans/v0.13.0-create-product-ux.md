## Problem

Marking allergens “free” is tedious; bilingual names are awkward; entering macros as raw numbers is hard to reason about.

## Goal

Faster, clearer product create/edit: bulk allergen free, optional NL/EN names, slider-based macros tied to kcal and weight.

## In scope

- Button: set all remaining allergens to `free` (does not clear `contains` already set)
- Optional `name_nl` + `name_en`; primary field follows UI language
- Macro sliders on create: kcal first; carbs/protein/fat share energy; grams capped by product weight (100 g basis); show g + kcal

## Out of scope

- Restaurant quick-log % sliders (v1.3)
- Goal g/kg sliders (v1.0.4)

## Acceptance criteria

- [ ] One tap sets remaining allergens to free; contains stays
- [ ] Can save with only primary language filled; second language optional
- [ ] Sliders cannot assign more than 100 g of a macro per 100 g product
- [ ] nl/en via i18n
