## Problem

Only day-level totals; hard to see progress toward goals while logging. Reports day view shows meal macros but not whether a meal was “too big” vs a fair share of the day goal.

## Goal

See macros **per meal** and **progress toward daily goals**. First slice: **reports day view** progress bars (selected macro only). Diary meal bars remain for later in this patch.

## Locked decisions (reports day progress — now)

| Decision | Choice |
|---|---|
| Where | **Reports day view only** (not diary yet) |
| Bars | Day total + each meal row that has food; **selected macro only** |
| Goal source | `goal_revisions` for that day; if none, fall back to live `profiles.goal_*` |
| Meal split | Deduct snack totals from day goal; remainder ÷ 3 = shared scale for all meal/snack bars. If snacks ≥ day goal → scale = dayGoal ÷ 3 |
| Shared meal scale | `trackMax = max(mealGoal, peak meal/snack)` so bars are comparable |
| Day total bar | Expands when over; **goal marker line**; strong red when over |
| Meal/snack bars | **No** goal marker. Soft coral only when **day total > day goal** AND that slot **> mealGoal**. If day ≤ goal → all green (even a huge single meal) |
| Empty mains | Always show breakfast/lunch/dinner (0 + grey bar); snacks only when logged |
| Missing goal | Hint to set in Settings; bars use default 2000 kcal + logical macros |
| Busy UI | Full-width day bar card; simple “Doel per maaltijd” (no snack text) |

### Meal scale examples (kcal day goal 2100)

| Snacks that day | Snack sum | Meal scale |
|---|---|---|
| none | 0 | 700 (2100/3) |
| one snack 300 | 300 | 600 ((2100−300)/3) |
| snacks ≥ 2100 | ≥2100 | 700 (fallback day/3) |

All shown bars (3 mains + any logged snacks) share that meal scale.

## In scope (full v1.0.5)

- Per-meal macro totals (diary)
- Daily progress UI for kcal + macros (respect count direction + focus/overview)
- Reports day: selected-macro progress vs day goal + per-meal share (this slice)

## Out of scope
