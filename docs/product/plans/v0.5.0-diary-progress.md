## Problem

- Day header (focus + overview) works; **per-meal totals look messy**.
- Diary meal progress bars would be too busy (Reports already has them).
- Need a clear **day footer** on Dagboek (kcal vs goal, distribution, allergens, room for micros later).

## Goal

1. Cleaner **per-meal totals** (receipt kcal + subtle C/P/F; keep macros visible for diabetes / gym use cases).
2. **Diary day footer** in a fixed module order (below).
3. **Ignore this day in statistics** checkbox (works from day one).
4. **Reports snack order** chronological between mains.

**Not in this patch:** diary per-meal progress bars; Reports pie; expand-macros-on-tap (focus mode already covers deep dive).

### Meal card totals (overview) - locked UI

```
[ foods… ]

  C g / P g / F g                    ← muted, small, slash-separated
                         Σ xxx g
                    842 kcal         ← receipt / calculator weight, right-aligned
```

- Focus mode: keep calm (existing focused total is fine; do not add noisy C/P/F if it fights the focus header).
- Do **not** use big three-box macro tiles.

### Diary day footer - locked order (top → bottom)

When the day has **at least one food entry**:

1. **Day kcal vs goal** - thin bar and/or `1820 / 2200`
2. **Ignore this day in statistics** - checkbox (see data note below)
3. **Allergen hits today** - only if profile has allergens selected: **count** + **gram %** of logged food that hits
4. **Macro pie** + **C / P / F grams legend**
5. **Micronutrients** - placeholder section (“soon” / grey rows); no fake numbers

When the day is **empty**:

- Hide pie / allergen / micro blocks
- Show **one-line coach**: e.g. “Log breakfast to see your split”
- Still show **Ignore this day in statistics** (user may mark vacation / sick day with no food)

### Ignore day in statistics

- Per-user, per calendar day flag (e.g. `diary_day_prefs.ignored_in_stats` or equivalent).
- When checked: day is excluded from Reports averages / week charts / any “statistics” aggregates that ship now or next (wire **already** in this patch so Reports respects it).
- Does **not** delete diary entries; only excludes from stats.
- Needs a small migration + RLS (own rows only).
