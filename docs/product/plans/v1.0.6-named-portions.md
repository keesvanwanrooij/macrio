## Problem

Products often need several natural sizes (small / medium / large apple), not one default portion.

## Goal

Log with multiple named portions per product version (labels + g/ml).

## In scope

- Picker UX when logging
- Seed a few Dutch staples with S/M/L where useful

## Out of scope

- Parent catalog (v1.0.8)
- User-defined portion templates across all products

## Acceptance criteria

- [ ] Units g/ml still work
