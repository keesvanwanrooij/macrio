## Problem

Only kcal + carbs/protein/fat blocks deeper goals (fiber, salt, micros) before launch.

## Goal

Store and show **extended nutrition** before public release: fiber, sugars, salt/sodium, saturated + unsaturated fat, plus **micronutrients** (vitamins/minerals), with honest unknowns.

## In scope

- Schema on versions + diary snapshot columns (or jsonb block) for new nutrients
- Create / edit / product / log-entry / diary / reports surfaces as needed
- OFF / seed mapping where tags exist; never invent `free`/zero for allergens; micros missing = unknown/null
- nl/en labels

## Out of scope

- Lab-grade certification
- Paying for proprietary nutrient DBs (prefer OFF + seed + community)

## Acceptance criteria

- [ ] User can enter and log extended fields
- [ ] Historical diary entries without new fields stay valid
- [ ] Seed/OFF path documented

## Open questions

- Exact micronutrient list for v1 (prioritize common EU label set)
- Display density on diary (overview vs detail screen)
