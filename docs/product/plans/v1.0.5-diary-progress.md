## Problem

Only day-level totals; hard to see progress toward goals while logging. Reports day view shows meal macros but not whether a meal was “too big” vs a fair share of the day goal.

## Goal

See macros **per meal** and **progress toward daily goals**. First slice: **reports day view** progress bars (selected macro only). Diary meal bars remain for later in this patch.

## Locked decisions (reports day progress — now)

| Decision | Choice |
|---|---|
| Where | **Reports day view only** (not diary yet) |
| Bars | Day total + each meal row that has food; **selected macro only** |
| Goal source | `goal_revisions` for that day; if none, fall back to live `profiles.goal_*` |
| Meal split | Always 3 mains × **2 parts**; each **logged** snack slot × **1 part**. No food in a snack slot → slot omitted (no bar). No snacks logged → each main = 1/3 of day goal |
| History bug | Accept for now: no past revision ⇒ treat like “current goal”; watch after goal changes mid-week |
| Busy UI | Thin progress bars under selected total cell + under meal values; no four-macro meal bars |

### Meal split examples (kcal day goal 2100)

| Logged slots | Parts | Breakfast | Snack after breakfast |
|---|---|---|---|
| 3 mains only | 6 | 700 (2/6) | — |
| 3 mains + 1 snack | 7 | 600 (2/7) | 300 (1/7) |
| 3 mains + 2 snacks | 8 | 525 (2/8) | 262.5 (1/8) |

Snack-when-logged is **not** heavy: count distinct snack slots with entries, then `goal * parts / totalParts`.
