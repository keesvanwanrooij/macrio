## Problem

Deleting diary rows is clumsy; recents forget custom grams; picking another product version at log time is buried; favorite staples are hard to find in search.

## Goal

Faster daily logging: swipe delete, restore last portion size, pick version on the portion screen, star favorites for search filter, optional search synonyms on versions.

## In scope

- Swipe-to-delete on diary entries (safe confirm or undo)
- Recents → log-entry prefills **last grams** for that user+product (including custom 130 g etc.)
- Version selector on log-entry; sort **likes → completeness score → newest**; prefer last version this user logged when available (see [`version-completeness-ranking.md`](version-completeness-ranking.md))
- Shared `versionCompletenessScore` helper (allergens filled `contains`/`free`; cook weights added when v1.0.7 ships)
- Star/favorite products; filter favorites in Search tab
- **Search synonyms:** version owner sets primary name + optional synonyms; search matches both

## Out of scope

- Duplicate-from-version-X on product page (v1.2)
- Community meal share (v1.4)

## Acceptance criteria

- [ ] Swipe delete removes entry; no accidental mass deletes
- [ ] Recent with 130 g opens log-entry at 130 g
- [ ] Can switch version before saving; ranking = likes → completeness → newest; last-used remembered next time
- [ ] Equal likes: version with more allergens set (contains/free) ranks above all-unknown
- [ ] Favorites filter in search shows only starred items
- [ ] Synonyms: search finds a version via primary name or an owner-set synonym
- [ ] nl/en via i18n
