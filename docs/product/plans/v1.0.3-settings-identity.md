## Problem

Username typos are stuck; DOB is awkward to type; no app version for support; date format should be a preference. Jumping many days in diary/reports with only ‹ › is slow.

## Goal

Users can fix username, pick dates with a native picker (DOB + diary/reports), choose date format, and see the app version in Settings.

## Locked decisions (diary / reports jump)

| Decision | Choice |
|---|---|
| Where | Diary **and** reports (same control) |
| Week mode | Jump to the **week that contains** the picked day (stay in week mode) |
| UI | Native system date picker (not custom month calendar) |
| Limits | Any past date + **today**; no future |

## Storage / how far back?

**Today:** `diary_entries` have **no retention limit**. Rows stay until the user deletes them or deletes the account. Indexed by `(user_id, date)`.

**Size:** Each entry is small (macros snapshot + FKs). Rough order: thousands of rows per user-year is still tiny vs product photos / community data. Allowing pickers to go **more than 1 year** does **not** change storage by itself: storage grows with logging, not with how far the UI can navigate.

**Product stance for v1.0.3:** allow any past date. Empty days simply show empty diary/reports. Do **not** invent auto-purge for this feature. If we ever need retention (GDPR policy, cost at huge scale), that is a separate product decision later, not a picker constraint.

## In scope

- Edit username (unique + format rules)
- App version in Settings
- Shared native date picker helper (one dependency for DOB + diary/reports)
- DOB picker (onboarding + profile)
- Tap date label on diary + reports → native picker → set anchor date
- Date format setting applied to displayed/entered dates

## Out of scope

- Full name changes (already optional elsewhere)
- Changing email (unless already trivial via Supabase)
- Custom in-app month calendar UI
- Automatic deletion / archival of old diary rows

## Acceptance criteria

- [ ] Username update fails clearly on collision / invalid format
- [ ] DOB picker works on Android (Expo Go)
- [ ] Diary: tap date → picker → opens that day (not future)
- [ ] Reports day: tap date → that day; reports week: tap date → week containing that day
- [ ] Date format setting persists on profile
- [ ] Version string matches `app.json` / Expo config
- [ ] nl/en via i18n

## Open questions

- Which date formats to offer (DD-MM-YYYY, YYYY-MM-DD, MM-DD-YYYY)?
- Minimum DOB / age for goal calculator (separate from diary jump)?

## Implementation notes

- Prefer `@react-native-community/datetimepicker` (Expo-compatible) behind a small shared component/hook used by Settings DOB, diary, and reports.
- Reports week: set `anchor` to picked date; existing `weekStart` logic already derives Monday week.
