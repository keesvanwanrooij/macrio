## Problem

Without locked parents, community products scatter and the DB becomes a mess before launch.

## Goal

System **parent** foods + Dutch-market staple seed covering ~80% of everyday eating; community children link to parents (or stay parentless when needed).

## In scope

- Data model: system parents; optional parent link on children
- Seed important Dutch parents (macros, allergens, portions, cook_state)
- Premium-AI fill session using the pitch file; human review before merge
- Pre-public launch blocker

## Out of scope

- Trust graph / version auto-trim (v1.2)
- NEVO import (not a goal)
- Users creating new parents in v1.0

## Acceptance criteria

- [ ] Users cannot create parents via normal create-product
- [ ] Users can link a new product to a parent or leave parentless
- [ ] Seed covers agreed staple list after founder review
- [ ] Allergen honesty preserved (`unknown` never coerced to `free`)
- [ ] Docs / SETUP mention running the new seed migration(s)
