## Problem

Macros differ a lot between raw and prepared weight (rice, meat, pasta). “100 g rice” is ambiguous.

## Goal

Every product/version declares `cook_state`: `cooked` | `uncooked` | `not_applicable`, visible when creating and logging.

## In scope

- DB column (prefer **product_versions** — pick one place and document)
- UI selector on create/edit
- Visible in search/log
- Update staples that need it; `not_applicable` for drinks/oils/etc.

## Out of scope

- Automatic raw↔cooked conversion calculator
- Parent catalog itself (v1.0.8)

## Acceptance criteria

- [ ] Migration + types updated
- [ ] Create-product requires or defaults cook_state sensibly
- [ ] Search/log shows cooked vs uncooked clearly (nl/en)
- [ ] Existing rows backfilled (likely `not_applicable` or best-effort)
