## Problem

Before public users: need product analytics and defenses against forgot-password spam (username trolling), without waiting for a post-launch major.

## Goal

Ship success-metrics analytics (privacy-aware) and server-side reset abuse controls (rate limits, cooldown, backoff, jitter), with optional CAPTCHA/honeypot/ops blocks in the same patch (can be gated).

## Acceptance criteria

- [ ] Analytics wired for metrics in `SUCCESS_METRICS.md` (no secrets/PII in git or public docs)
- [ ] Forgot-password requests rate-limited (IP + account + spray); cooldown + progressive IP backoff + light jitter
- [ ] Escalate path documented/implemented: CAPTCHA or Turnstile + honeypot; ops alert / temp block
- [ ] Copy stays generic (no account enumeration)
- [ ] Distinct from Sentry crash SDK (**v0.1.0**)

## Out of scope

- Store listing / custom SMTP / production Site URL (remain **v1.0.0** ops)
- Trust graph, workouts, community (post-public majors)
