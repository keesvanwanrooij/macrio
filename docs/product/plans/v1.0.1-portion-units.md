## Problem

Liquids show as grams. Custom amount often starts from an awkward default instead of a natural 100 g / 100 ml.

## Goal

Users see **ml** for drinks and get a **100** default when entering a custom amount.

## In scope

- `unit: 'g' | 'ml'` on portion objects in JSONB
- Seed updates for liquid staples
- UI shows ml where `unit === 'ml'`
- Default custom amount to 100 (g or ml matching unit)
- Fractional named-portion counts (step ±0.5; free-typed fractions; e.g. half an AH ovenschotel)
- Macro math: keep per-100g internally; 1 ml ≈ 1 g for water-like liquids for now

## Out of scope

- Full product-level reference unit / nutrition per 100 ml (later patch)
- Oils and dense liquids accuracy

## Acceptance criteria

- [ ] Liquid seed items show ml in portion picker and product UI
- [ ] Custom amount defaults to 100
- [ ] Can log 0.5 (and other) × a named portion; macros scale correctly
- [ ] nl/en via i18n
- [ ] No migration required if portions stay JSONB-only
