## Problem

Users cannot see at a glance how trustworthy a version is, and improving gaps is not guided.

## Goal

1. Visible **quality badge** on versions (star tiers).
2. On open: **Help dit product verbeteren** with checklist of missing fields (Wikipedia-style).

## Badge examples (copy TBD nl/en)

| Tier | Example label |
|---|---|
| ★★★★★ | Verified |
| ★★★★ | Community Trusted |
| ★★★ | Good |
| ★★ | Needs Review |
| ★ | Incomplete |

## Inputs (examples)

- Macros filled
- Allergens complete (not all unknown if profile expects honesty - define carefully)
- Barcode present
- Like count / distinct users
- Age of version
- Open reports
- Multiple confirmations (likes as confirm)

## Missing-info checklist (examples)

- Ingredients
- Allergens
- Portions
- Cooked / uncooked
- Photo

## Acceptance criteria

- [ ] Badge visible on product/version UI
- [ ] Checklist shows only truly missing items; taps route to suggest-edit / create version where possible
- [ ] Ranking/completeness helpers share one scoring story where practical
