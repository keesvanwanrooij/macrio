## Problem

Without crash monitoring, launch bugs on real devices are invisible except vague in-app feedback.

## Goal

Ship **Sentry** (or equivalent) before public users so crashes and selected errors are actionable.

## In scope

- SDK in Expo/RN app; DSN from env; no secrets in git
- Source maps for release builds
- Basic PII scrubbing (no auth tokens / emails in events by default)
- Dev test hook or documented way to verify

## Out of scope

- Product analytics / funnels (**v0.15.0**)
- Full session replay unless trivial later

## Acceptance criteria

- [x] Release build can report a test crash/error to the dashboard (Expo Go: Settings smoke button with DSN)
- [x] Documented in SETUP or ops notes for the founder

## Shipped notes

- Package: `@sentry/react-native`; config: `app/app.config.js` + `app/metro.config.js`
- Runtime: `app/src/lib/sentry.ts`; root wrapped in `_layout.tsx`
- Optional `EXPO_PUBLIC_SENTRY_DSN`; `SENTRY_AUTH_TOKEN` only for future EAS source-map upload
