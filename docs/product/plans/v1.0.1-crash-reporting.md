## Problem

Without crash monitoring, launch bugs on real devices are invisible except vague in-app feedback.

## Goal

Ship **Sentry** (or equivalent) before public users so crashes and selected errors are actionable.

## In scope

- SDK in Expo/RN app; DSN from env; no secrets in git
- Source maps for release builds
- Basic PII scrubbing (no auth tokens / emails in events by default)
- Dev test hook or documented way to verify

## Out of scope

- Product analytics / funnels (v1.5)
- Full session replay unless trivial later

## Acceptance criteria

- [ ] Release build can report a test crash/error to the dashboard
- [ ] Documented in SETUP or ops notes for the founder

## Open questions

- Sentry vs another vendor if pricing/privacy prefers it (default: Sentry)
