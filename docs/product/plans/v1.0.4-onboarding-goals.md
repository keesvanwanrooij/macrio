## Problem

Calculator dominates goals; allergen “no allergies” is unclear; body fields can be nonsense; macro targets are hard for non-standard diets.

## Goal

Calmer onboarding: calculator on demand, clear “no allergies”, safe body limits, smarter macro targets with kcal hints.

## In scope

- Calculator hidden by default; button “Berekenen op basis van je lichaam” / EN
- **Niets / Nothing** = skip allergies → `allergens: []`
- Soft validation height/weight/age (from DOB), non-shaming errors
- Leave one macro empty → autocomplete from kcal + other two; show kcal equivalents

## Out of scope

- Full Yazio-style questionnaire (v1.3)
- Changing Mifflin formula fundamentally

## Acceptance criteria

- [ ] Calculator expands only after button press
- [ ] Niets leaves empty allergen list and continues
- [ ] Out-of-range body values blocked with friendly copy
- [ ] Autocomplete + kcal equivalents on onboarding and Settings
- [ ] nl/en via i18n
