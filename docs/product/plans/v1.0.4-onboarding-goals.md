## Problem

Calculator dominates goals; allergen “no allergies” is unclear; body fields can be nonsense; macro targets are hard for non-standard diets; raw gram targets are hard to relate to body weight.

## Goal

Calmer onboarding: calculator on demand, clear “no allergies”, safe body limits, smarter macro targets with kcal hints and **g/kg sliders**.

## In scope

- Calculator hidden by default; button “Berekenen op basis van je lichaam” / EN
- **Niets / Nothing** = skip allergies → `allergens: []`
- Soft validation height/weight/age (from DOB), non-shaming errors
- Leave one macro empty → autocomplete from kcal + other two; show kcal equivalents
- **Macro goal sliders** (onboarding + Settings): kcal set or calculated; protein/carbs/fat on logical **g/kg** ranges (e.g. protein ~0.8–2.2 g/kg); show **grams and kcal** for each macro

## Out of scope

- Full Yazio-style questionnaire (v1.3)
- Changing Mifflin formula fundamentally
- Create-product macro sliders (v1.0.9)

## Acceptance criteria

- [ ] Calculator expands only after button press
- [ ] Niets leaves empty allergen list and continues
- [ ] Out-of-range body values blocked with friendly copy
- [ ] Autocomplete + kcal equivalents on onboarding and Settings
- [ ] g/kg sliders update gram + kcal displays; persist to profile goals
- [ ] nl/en via i18n
