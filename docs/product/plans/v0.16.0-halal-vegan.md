## Problem

Users who eat **halal** or **vegan** need the same honest, glanceable signals as allergen pills. Missing data must stay unknown. Veganism on a product is not a single yes/no: contributors need a simple ladder from standard vegan → stricter vegan → not vegan.

## Goal

Add **Halal** and **Vegan** as the last pills on the allergen chip row (after EU-14). Profile toggles control whether those signals matter in search (same pattern as allergens).

## States & copy (locked)

### Halal (3-way cycle)

Tap cycle: `unknown` → `yes` → `no` → `unknown`

| Value | EN (chip / long) | NL (chip / long) |
|---|---|---|
| `unknown` | Halal unknown | Halal onbekend |
| `yes` | Halal | Halal |
| `no` | Not halal | Niet halal |

### Vegan (4-way cycle: simple → strict → not → unknown)

Tap cycle: `unknown` → `vegan` → `strict_vegan` → `not_vegan` → `unknown`

| Value | EN | NL | Meaning on a **product version** (not lifestyle motivation) |
|---|---|---|---|
| `unknown` | Vegan unknown | Vegan onbekend | No reliable claim yet (default) |
| `vegan` | Vegan | Vegan | Standard dietary vegan: no meat, fish, dairy, eggs, honey (or other obvious animal ingredients) as declared |
| `strict_vegan` | Strict vegan | Strikt vegan | Stricter claim: also no animal-derived additives / processing aids when the contributor asserts that; use only when reasonably sure, else leave `vegan` or `unknown` |
| `not_vegan` | Not vegan | Niet vegan | Contains animal products |

No “may not be vegan” / may_contain-style middle state for vegan or halal.

## Profile preferences

- Settings + onboarding (as fits existing allergen UX): **I eat halal**, **I eat vegan**
- If unset: hide diet warnings/filters in search (same as allergens with empty selection)
- If **I eat vegan**: treat both `vegan` and `strict_vegan` as OK; warn on `not_vegan` and `unknown` (same honesty pattern as allergens)
- If **I eat halal**: warn on `no` and `unknown`

**Not the allergen “Niets” collapse:** EU-14 uses a leading **Niets/Nothing** chip that clears the multi-select and hides option pills until tapped again (`noneChipListState` in `allergens.ts`). Halal/vegan profile prefs are **independent booleans** (“I eat …”) on the same chip row. Do not add a second Niets for diet. Unset diet prefs already mean “no warnings” without collapsing the row.

## UI

- Same chip row as allergens: EU-14 → Halal → Vegan (last)
- Short names on chips; longer labels in lists/diary/product where allergens already use long form
- Legend / help: one line explaining Halal triad + Vegan ladder

## Data & OFF

- Store on **product versions** (snapshot rules follow existing product/version model)
- Default / missing → `unknown`
- Open Food Facts: map known vegan/halal tags when present; otherwise **unknown** (never invent)
- **Admin staple seed (with v0.11 / v0.14):** curated seed versions must include correct macros, micros, EU-14 allergens, **and** halal + vegan where the food is known. Completing diet labels on seed is part of this patch’s pre-launch bar.

## Acceptance criteria

- [ ] Create/edit cycles Halal and Vegan correctly; defaults unknown
- [ ] Search/list behavior respects profile toggles like allergens
- [ ] nl/en strings complete
- [ ] OFF import does not set yes/vegan without evidence
- [ ] No may-contain equivalent for these two

## Out of scope

- Vegetarian-only label, kosher, certifications marketplace
- Inferring vegan from allergen tags alone (too error-prone; OFF/explicit only)
- Motivation categories (ethical / environmental vegan) as separate product fields
