## Problem

Calculator dominates goals; allergen “no allergies” is unclear; body fields can be nonsense; macro targets are hard to relate to body weight.

## Goal

Calmer onboarding: calculator on demand (default closed), clear “no allergies”, soft height/weight limits, synced g/kg + gram macro controls with kcal cascade.

## Locked decisions

| Topic | Choice |
|---|---|
| Soft body limits | Height **100–230** cm and weight **30–250** kg get friendly messages. Age **under 16** always shows a trusted-person soft message (still calculates). Block numeric fields **≤ 0**. |
| Macro model | Soft g/kg rails: protein **0.8–2.2**, fat **0.5–1.5**, carbs **3.0–10.0**. Carbs mostly fill kcal. |
| Cascade | **kcal →** recalculate macros. **protein/fat →** carbs fill rest. **carbs →** update kcal. Typed grams may leave soft rails; sliders still show the value. |
| Weight gate | Need weight for g/kg sliders; else gram-only. Warn if macro kcal vs goal kcal beyond **5%**. |
| UI | Sliders + always-visible gram fields, two-way sync, clean layout, kcal equivalents. |
| Autocomplete | When one of P/C/F is empty and **another field changes**, fill the empty one with cascade math. |
| Allergens | **Niets** and **Skip** both persist `allergens: []`. |
| Calculator | “Berekenen op basis van je lichaam”; **default closed** onboarding + Settings. Sliders when weight exists. |
