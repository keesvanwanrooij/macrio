## Problem

EU users and store review require in-app **delete account** and a path to **access/export** personal data. Missing this risks rejection at submission.

## Goal

One Settings privacy flow: **download my data** + **delete my account** (with clear confirmations).

## In scope

- Export personal data (profile, diary entries, goals, feedback metadata as applicable) as downloadable CSV and/or JSON
- Delete account: auth user + cascade/anonymize per DATA_MODEL (products may remain anonymized for community)
- nl/en copy; irreversible delete confirmation
- Align with LEGAL.md disclaimers where needed

## Out of scope

- Separate “power user diary CSV only” product (export is this compliance download)
- Admin bulk tools

## Acceptance criteria

- [ ] Signed-in user can export data without emailing support
- [ ] Signed-in user can delete account in-app; cannot sign in afterward
- [ ] Store guideline checklist documented for submission

## Open questions

- Exact tables in export v1
- Soft grace period vs immediate hard delete
